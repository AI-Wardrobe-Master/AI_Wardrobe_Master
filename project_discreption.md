# AI Wardrobe Master — Project Brief (Phase 1)

## 0. One-liner
一个“衣物数字资产管理 + 平台内容导入”的手机应用：把衣服拍成结构化 **Asset Card**，自动打标签、可检索、可分衣柜管理；同时支持 Creator 打包分享衣物/搭配，用户一键导入到“虚拟衣柜”并做 **2.5D 试穿式搭配预览**。

---

## 0.1 Tech Stack (Current)
- **Client**：Flutter（Dart），仓库主应用目录：`ai_wardrobe_app/`
- **Target**：iOS / Android（可按需扩展 Web，用于调试或演示）

---

## 1. Project Goal & North Star
- **核心目标**：提高衣物复用率与搭配效率，让“衣物”从静态物品变成可检索、可组合、可复用的数字资产。
- **North Star Metric**：每周被有效复用的衣物资产数量（effectively reused clothing assets / week）。

---

## 2. Scope Boundary (Phase 1)

### 2.1 In Scope (必须做)
1) **衣物数字化**：拍摄正反面 -> 去背景/主体抠图 -> 生成多角度（每 45°）视觉 -> 存为 Asset Card  
2) **结构化识别与搜索**：自动标签（品类、颜色/图案、风格、季节、人群等）+ 置信度 + 支持手动纠错 + 基于标签/关键词的快速检索  
3) **多衣柜管理**：可创建多个衣柜；衣物实体不重复存储，通过引用/归属关系管理  
4) **平台内容交互**：
   - Creator 侧：创建/编辑 Card Pack（衣物包/搭配包），生成分享/分发链接
   - User 侧：一键导入 Card Pack 到“虚拟衣柜”，并与自有衣物 **来源隔离**（避免数据污染 + 支持溯源 Provenance Tracking）
5) **2.5D 搭配可视化**：虚拟模特/画布，支持从（自有 + 虚拟）衣柜拖拽组合，生成搭配预览；可保存/命名/回看历史搭配

### 2.2 Out of Scope (Phase 1 明确不做)
- 身材扫描/医学级合身度诊断
- 交易闭环：支付、履约、售后
- 复杂版权系统（授权/撤回/侵权处理闭环）

---

## 3. Users & Key Flows

### 3.1 Users
- **普通用户（Consumer）**：管理自有衣物 + 导入 Creator 内容 + 搭配预览
- **内容创作者/主播（Creator）**：上传衣物资产与搭配集合，打包分发给用户导入

### 3.2 Core User Flows
1) Add Clothing (Owned)
   - 拍摄正/反面 -> 去背景 -> 自动分类+标签 ->（可编辑）-> 入库 -> 加入衣柜
2) Search & Filter
   - 标签/关键词检索（<1s）-> 过滤：场景/颜色/来源（Owned vs Imported）-> 进入详情卡片
3) Import Creator Pack (Virtual Wardrobe)
   - 打开链接 -> 导入衣物/搭配 -> 自动标记“未拥有” -> 来源隔离 + 溯源信息保留
4) 2.5D Outfit Visualization
   - 从两个来源衣柜选衣 -> 画布拖拽/叠加 -> 生成预览 -> 保存/命名/删除

## 4. WBS → Module Map (Updated)

### 1.0 Clothing Digitalization and Registration
- 1.1 使用移动设备摄像头捕捉服装的前视图和后视图图像
  - 1.1.1 开发摄像头集成以实现图像捕捉序列
  - **(Removed)** 1.1.2 预览、确认和重拍功能（deleted）
  - 1.1.3 处理图像质量检查和用户反馈
- 1.2 生成去背景表示
  - 1.2.1 集成图像处理算法进行背景移除
  - 1.2.2 在服装记录中存储原始图像和处理后的图像
  - **(Removed)** 1.2.3 优化移动设备性能（deleted）
- 1.3 生成以45度间隔的多角度视觉表示
  - 1.3.1 开发图像生成模型或算法进行角度旋转
  - 1.3.2 处理并存储一系列角度特定的图像
  - 1.3.3 确保跨角度的图像质量一致性
- 1.4 在持久化数据库中存储服装记录
  - 1.4.1 设计服装数据对象的数据库架构
  - 1.4.2 实现数据持久化和检索机制
  - 1.4.3 确保应用重启后的数据完整性

### 2.0 Automated Classification and Search Support
- 2.1 自动将服装物品分类到预定义类型
  - 2.1.1 集成AI/ML模型进行基于图像的分类
  - 2.1.2 生成并显示类型标签，附带置信度指示器
  - 2.1.3 处理未确认或低置信度分类
- 2.2 分配颜色和图案属性
  - 2.2.1 开发颜色检测和图案识别算法
  - 2.2.2 在服装记录中填充并存储属性
  - 2.2.3 允许用户修改分配的属性
- 2.3 支持额外的可配置服装属性（例如，用户群体、风格、季节）
  - 2.3.1 定义并实现属性类别
  - **(Removed)** 2.3.2 基于图像分析自动进行初始分配（deleted）
  - 2.3.3 提供手动调整的编辑界面
- 2.4 允许手动确认或更正分类结果
  - 2.4.1 构建用于审查和编辑的用户界面
  - 2.4.2 确保用户更改持久化覆盖自动化结果
  - 2.4.3 使用分类属性集成搜索支持

### 3.0 Wardrobe Management
- 3.1 创建、重命名和删除衣柜
- 3.2 从衣柜中添加或移除服装物品
- 3.3 支持用于平台来源物品的虚拟衣柜

### 4.0 Platform Content and Outfit Collections
- 4.1 Creator 上传服装物品
- 4.2 创建并发布搭配集合
- 4.3 浏览 Creator 与发布内容
- 4.4 将内容保存到虚拟衣柜

### 5.0 Outfit Visualization
- 5.1 从衣柜中选择服装物品进行可视化
- 5.2 查看、保存或删除可视化结果

---

## 5. Definition of Done (Phase 1)
- 每个模块能跑通最小闭环：Add clothing → tags → search → wardrobe → import pack → 2.5D preview
- 数据模型具备：来源隔离 + 溯源字段（Provenance）
- 删除/导入/衣柜操作不会破坏衣物实体一致性
