Missing Backend Features - Architecture Diagram

架构重点是把新增能力放到现有 FastAPI + SQLAlchemy + Storage + Worker 的骨架内。 导入和 Outfit 作为新的聚合根处理，Creator Content 作为平台域能力补入，Virtual Wardrobe 则由导入服务托管，而不是交给前端保证。

Client Layer

Consumer App

浏览衣物、搜索、导入 Pack、查看 Virtual Wardrobe、保存 Outfit。

Creator Console

上传 Creator Item、维护 Pack、发布与归档、查看导入统计。

API Layer

Clothing API

扩展上传校验，继续负责衣物详情、搜索和标签确认。

Creator / Card Pack API

新增 Creator Profile、Creator Item、Card Pack CRUD 与发布接口。

Import / Outfit API

新增导入事务、导入历史、Outfit CRUD 与缩略图触发接口。

Domain Services

Upload Validation Service

统一处理 MIME、扩展名、大小、空文件、像素尺寸约束。

Creator Content Service

负责 Creator Item 生命周期、Pack 组装、分享标识和发布状态机。

Import Service

负责 Pack 校验、IMPORTED 副本生成、provenance 填充和导入事务。

Virtual Wardrobe Service

负责系统虚拟衣橱创建、保护语义和与导入流程对接。

Outfit Service

负责布局持久化、归属校验、缩略图任务投递与详情拼装。

Infra Layer

PostgreSQL

落 users、creator_profiles、card_packs、import_histories、outfits 等表。

Object Storage

继续承载衣物原图、处理图、多角度、3D 模型、Pack Cover 与 Outfit Thumbnail。

Background Worker

继续处理 clothing pipeline，并新增 pack import 后置任务与 thumbnail 生成。

External AI / Renderers

沿用分类、多角度和渲染能力，不新增平行基础设施。

Consumer App / Creator Console
→
FastAPI Routers
→
Domain Services + Transaction Orchestration
→
DB + Storage + Worker

Core Aggregates And Tables

Identity

`users` 继续保留认证主表，`creator_profiles` 作为创作者公开资料扩展。

Clothing Domain

`clothing_items` 继续作为核心衣物实体，新增 provenance 字段支持 IMPORTED 来源追踪。

Wardrobe Domain

`wardrobes` 增加系统托管语义，`wardrobe_items` 保持用户域内的衣物归属关系。

Creator Content

`card_packs`、`card_pack_items` 负责平台内容聚合、顺序和发布状态。

Import Audit

`import_histories` 与 `import_history_items` 负责导入批次审计和 imported 副本映射。

Outfit Domain

`outfits` 与 `outfit_items` 负责持久化画布布局和缩略图状态。

Critical State Machines

CardPack.status DRAFT → PUBLISHED → ARCHIVED。只有 DRAFT 允许编辑条目集合和直接删除。
ImportHistory.status PENDING → IMPORTING → COMPLETED / FAILED。失败时必须整批回滚 imported 副本与 wardrobe_items。
Outfit.thumbnailStatus NOT_GENERATED → GENERATING → READY / FAILED。基础保存与缩略图生成解耦。
Clothing ingestion VALIDATED → PROCESSING → READY。校验失败不进入对象存储和异步任务。
