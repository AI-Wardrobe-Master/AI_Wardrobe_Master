Missing Backend Features - End-to-End Flow

这份流程图把现有评审文件中的缺口统一到四条业务主线里： 上传前置校验、创作者内容发布、消费者导入与溯源、以及 Outfit 保存。 设计目标是让新增能力尽量复用当前 `clothing -> pipeline -> storage` 的主链路，而不是额外再造一套平行系统。

入口约束：文件校验前置 导入策略：生成用户私有 IMPORTED 副本 系统资源：Virtual Wardrobe 由后端保障存在 引用边界：Outfit 只能引用当前用户域下的衣物

Flow 1. 创作者发布链路

目标是补齐 Creator Upload 与 Card Pack 生命周期，并复用现有服装处理流水线。

1

Creator 上传衣物

通过 Creator Item 接口提交正反面图、名称、描述和可选自定义标签。

→
2

API 前置校验

校验 MIME、扩展名、空文件、大小与像素尺寸，不合法请求直接返回 4xx。

→
3

复用现有处理流水线

存原图、去背、多角度、3D、AI 标签预测，生成 Creator Catalog 衣物资产。

→
4

创建或编辑 Card Pack

从创作者衣物里选择条目，形成 pack item 顺序和封面。

→
5

发布并生成分享标识

Pack 进入 PUBLISHED 状态，冻结分享入口和对外可预览数据。

CardPack: DRAFT CardPack: PUBLISHED CardPack: ARCHIVED

Flow 2. 消费者发现与导入链路

目标是补齐 Browse / Import / Provenance / Virtual Wardrobe 的后端闭环。

1

浏览 Creator 或分享链接

用户读取公开创作者资料、公开 Pack 列表，或直接通过 shareId 打开单个 Pack。

→
2

读取已发布 Pack 预览

后端只暴露 PUBLISHED pack，附带 cover、itemCount、creator 摘要与衣物预览。

→
3

执行 Import

Import Service 校验幂等键，拒绝导入非发布内容，并开启事务。

→
4

确保 Virtual Wardrobe 存在

系统托管衣橱首次导入时自动创建，后续重复复用，不依赖前端预先建好。

→
5

物化 IMPORTED 副本并落溯源

为当前用户生成私有 clothing item 副本，挂上 creator、pack、importHistory 与 importedAt。

→
6

写入衣橱与导入历史

批量写入 wardrobe_items、import_histories、import_history_items，再返回导入结果。

ImportBatch: PENDING ImportBatch: IMPORTING ImportBatch: COMPLETED / FAILED

Flow 3. 上传校验与标签修正链路

目标是让上传入口尽早失败，并保持 predictedTags 与 finalTags 的职责分离。

1

提交正反面图片

Consumer 或 Creator 通过统一上传入口提交衣物素材。

→
2

入口校验失败即返回

非法文件不会进入对象存储或异步处理链，错误码会明确区分类型与原因。

→
3

合法文件进入处理队列

生成 ProcessingTask，继续执行背景处理、多角度、模型与标签预测。

→
4

保存 predictedTags

系统产出的推荐标签不可变，用作审核参考，不直接作为最终搜索结果。

→
5

用户确认 finalTags

编辑后的 finalTags 才进入搜索、导入预览和 Outfit 选择结果。

Classification: UPLOADED Classification: CLASSIFIED Classification: USER_CONFIRMED

Flow 4. Outfit 保存与缩略图链路

目标是让画布保存成为稳定的后端资源，而不只是前端临时状态。

1

选择衣物并布置画布

用户从 OWNED 与 IMPORTED 两个来源的衣物中选择条目，形成布局状态。

→
2

保存 Outfit

Outfit API 校验衣物归属，持久化 outfit 与 outfit_items 的 layer / position / scale / rotation。

→
3

异步或幂等生成缩略图

Thumbnail Service 基于画布快照生成 preview，不阻塞基础保存成功。

→
4

支持列表 / 详情 / 编辑 / 删除

后端返回可复原画布的布局数据，供后续继续编辑或展示。
